# 読書記録アプリ タスク一覧

## 📋 概要

[specification.md](specification.md) および [design-doc.md](design-doc.md) に基づき、**紙本・電子籍の読書記録（CRUD、状態フィルタ、localStorage 永続化）**を Vite + React 19 + TypeScript で実装するためのタスクです。  
**メモアプリは上書き削除してよい**（合意）ため、併用・キー分離の議論は要らない。**ルーターなし**の一覧/詳細切替、**confirm 付き削除**、**`updatedAt` 降順**の既定表示を満たします。

**パッケージマネージャ**: 本リポジトリは `npm`（`package.json` の `scripts` を使用）。`pnpm` の記述は**使用しない**。

**テスト**（**2026-04-23 合意**）: **v1 は手動確認 + `npm run build` + `npm run lint` まで。Vitest は必須にしない**（v1.1 以降で検討）。旧・フェーズ 4 は**任意の拡張**。

## ✅ タスクリスト

### フェーズ 0: 入口・README

1.  **アプリの目的と学習用の一文を README に追記する**

    - 概要: **メモを読書に置換**する方針を **README** に 1 文以上で書く（学習用リポの位置づけと合わせてよい）。**読書用** `localStorage` キー名を**ドキュメントに明記**する
    - 完了条件:
      - [ ] 上記方針が「学習者が迷わない」レベルで分かる
      - [ ] 読書用キー名が文章で明示されている（旧メモキーは**残っても**利用しない旨があってもよい）
    - 動作確認:
      - [ ] 該当がドキュメントのみのため **`npm run build` / `npm run lint` に影響しない**（ドキュメントのみなら N/A も可）
    - メモ: コード変更が入るなら、必ず `npm run build` / `npm run lint` を通す

### フェーズ 1: ドメイン型と永続層

2.  **本（`Book`）の型と状態の定義**

    - 概要: `id`, `title`, `status`（未読/読書中/読了）, `notes`, `startedOn?`, `finishedOn?`（`YYYY-MM-DD` 相当の文字列 or null で統一）, `createdAt`, `updatedAt` の **`type` 定義**。日本語表示ラベルは UI 層のマッピングで扱う
    - 完了条件:
      - [ ] 型が単一のソース（例: `src/reading/types/book.ts` 等）に置かれている
      - [ ] `status` が列挙的に**誤代入しにくい**形（union 等）になっている
    - 動作確認:
      - [ ] `npm run build` が通る
      - [ ] `npm run lint` でエラーが出ない
      - [ ] 手動: 新規ファイルを import するダミー利用がなく、ビルドが通ること

3.  **`localStorage` 用の定数と読み取り/書き込み方針**

    - 概要: ストレージ**キー文字列**（`reading-journal:books` 等。旧メモキーは**使わない**）。読み取り失敗・JSON 壊損時は**空配列**＋**ユーザー向け短い警告**（バナー or コンソール＋1 行 UI）の方針を**どこに出すか**を決め、**フック実装**に回す
    - 完了条件:
      - [ ] キー名がコード上の 1 定数に集約されている
      - [ ] 壊損時の挙動が仕様と一致（空配列 + 通知）
    - 動作確認:
      - [ ] `npm run build` / `npm run lint` が通る
      - [ ] 手動: DevTools Application でキーを**一度壊した JSON**にしてリロードし、落ちずに**空＋通知**が出る（実装タスク 4 完了後の確認でも可）

4.  **`useBooks`（仮名）フック: 配列 state + デバウンス保存**

    - 概要: `add` / `update` / `remove` 相当、`books` 変更時に**150ms 前後のデバウンス**で `setItem`。追加・更新で `createdAt`/`updatedAt` を**仕様どおり**更新。表示用 **ソート**は `updatedAt` **降順**（保存配列を整列しても、表示用コピーでも可）
    - 完了条件:
      - [ ] リロード後、データが復元される
      - [ ] 連続更新で**過度に** `setItem` しない（デバウンス）
    - 動作確認:
      - [ ] `npm run build` / `npm run lint` が通る
      - [ ] 手動: 本を追加 → リロード → 残存。感想的に複数行編集 → 1 回の保存系で破綻しない

### フェーズ 2: UI コンポーネント

5.  **フィルタ（すべて / 未読 / 読書中 / 読了）＋ タイトル検索（v1）**

    - 概要: **Tab 操作で辿れる**セグメントボタンまたは `radio`+`label`。併せて**タイトル部分一致**の検索欄（空で全件）。**タグ**は**スコープ外**（[specification.md](specification.md) 合意）
    - 完了条件:
      - [ ] 切替で表示件数が一致する
      - [ ] 0 件のとき**空メッセージ**が出る
    - 動作確認:
      - [ ] `npm run build` / `npm run lint` が通る
      - [ ] 手動: 4 区分のデータを用意し、各フィルタの件数を目視

6.  **本一覧行（行クリックで選択、サマリ表示）**

    - 概要: タイトル、**状態**、**感想の1行サマリ**（長文は省略）、日付のあれば表示
    - 完了条件:
      - [ ] 行がキーボード**フォーカス**可能、または**論理的**に選択可能
      - [ ] 感想全文は**一覧に出さない**
    - 動作確認:
      - [ ] `npm run build` / `npm run lint` が通る
      - [ ] 手動: 狭い幅/広い幅で**読める**

7.  **新規登録 / 編集フォーム（詳細）**

    - 概要: `input` タイトル必須（空なら**無効**または**エラー**）、`select` または**同等の UI** で状態、**`input type="date"`** 2 箇所（空可）、`textarea` 感想、**保存**、**戻る**（狭幅用）、**削除**（**`window.confirm` 後**のみ削除）
    - 完了条件:
      - [ ] 各コントロールに**紐づく** `label` または `aria-*`
      - [ ] 削除は confirm 必須
    - 動作確認:
      - [ ] `npm run build` / `npm run lint` が通る
      - [ ] 手動: 日付空のまま「読了」にして**保存**できる

### フェーズ 3: 画面統合とレスポンシブ

    - 完了条件:

    - 動作確認:

      - [ ] 手動: ブラウザ幅 375px 付近と 1200px 付近で**主要導線**を確認

9.  **`App` を読書アプリのエントリにする（メモ置換）**

    - 完了条件:
      - [ ] ビルド成果物に**メモ専用の導線**が残っていない（意図した置換）
    - 動作確認:
      - [ ] `npm run build` / `npm run lint` が通る
